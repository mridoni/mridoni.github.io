October progress update

GixSQL v1.0.18b (still a pre-release - or ad interim if you prefer) is almost ready and should be out in a couple of days at most. As I am writing this, the packages are being built and, after some testing, will be published on GitHub. Then the work will start on v1.0.19. GixSQL v1.0.18b will include solutions for issues #119, #88 and PR #127 (thanks to Simon Sobisch for that).

The new build/packaging process (based on GitHub Actions) is going well, and should be ready for a pre-release in a couple of weeks at most, maybe less. As some of you may have noticed, a new branch (named "new-debugger") has debuted on Gix-IDE's repository. When everything is ready:

  • the "new-debugger" branch will be merged into the main branch
  • the workflow files for Github Actions will be adjusted and made public
  • binary packages will be released from the new build/packaging system

For the more adventurous of you: the "new-debugger" branch is already usable and should compile (almost) without problems, but your mileage, as usual, may vary.
